Проект - совокупность мероприятий направленных на создания уникального продукта, ограниченное по времени, бюджету, ресурсам.
Процесс - совокупность взаимосвязанных и взаимодействующих операций, преобразующих входные (исходные) данные в завершенный общий результат.
Каскадная модель - это система основанная на жёсткой последовательности процесса разработки: анализа требований, проектирования, реализации, тестирования, интеграции и поддержки.
Спиральная модель - процесс разработки программного обеспечения, сочетающий в себе итеративность и этапность.

Scrum - процесс разработки который разбивается на отдельные этапы, результатом каждого из которых является готовый продукт.
XP - это методология разработки ПО при которой осуществляется проверка одним программистом кода, написанного другим программистом,
где один программист занимается написанием кода, а его напарник в это же время непрерывно просматривает только что написанный код.
RUP - методология разделение проекта на несколько мелких проектов при котором каждая итерация разработки четко определена набором целей,
которые должны быть достигнуты в конце итерации. Такая методология существует с целью снижения рисков при работе с большим проектом
MSF учитывает постоянные изменения требований к конечному продукту, процесс разработки состоит из коротких циклов и представляет собой поступательное движение от простейших ранних версий продукта к его окончательному виду.

Диаграмма ганта - это визуализация плана разработки проекта, содержащий задачи и сроки их выполнения.
Применяется для организации налаженной разработки ПО, где проблемы могут быть обнаружены и устраненные на ранней стадии
Требования к проекту - это набор задач, предъявленные к исполнителю, которые должны быть выполнены для достижения завершения проекта и получения заранее оговоренной суммы выплаты.

Программа разработки диаграммы ганта.
ТЗ:
Программа должна содержать панель просмотра диаграммы.
Панель просмотра должна представлять собой таблицу, где будут такие столбцы как номер задачи, её название,
срок начала и окончания, а так же столбцы разделения задачи на дни, недели или месяца (с зависимости от режима просмотра) на выполнения этой задачи.
Сама задача это прямоугольник который проходит через один или несколько столбцов-периудов задачи.

Возможность создания, удаления, и редактирования задачи. В частности изменение названия и сроков.
Возможность изменять режим просмотра временной шкалы задач: по часовой, по дневной, по недельной.
Учитывание количетва рабочих дней и рабочего времени.

Возможность изменение срока задачи системой перемещения и/или растягивания объекта-задачи (представленного прямоугольником) по графику по горизонтале

Возможность сохранять таблицу. А так же открывать её с помощью этой программы.
Скролинг но горизонтале для просмотра полной информации о всех времменых ограничений задач
